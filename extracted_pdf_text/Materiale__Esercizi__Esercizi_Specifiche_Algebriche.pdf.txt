===== PAGE 3 =====
Esercizio 1 (2/3)
declare: s, s1, s2: String, c: Char
isEmpty(new()) = true;
isEmpty(add(s, c)) = false;
length(new()) = 0;
length(add(s, c)) = length(s) + 1;
append(s, new()) = s;
append(s1, add(s2 ,c)) = add(append(s1, s2), c);
equal(new(), new()) = true;
equal(new(), add(s,c)) = false;
equal(add(s, c), new()) = false;
equal(add(s1, c),add(s2, c)) = equal(s1, s2);

===== PAGE 14 =====
Soluzione Esercizio 3
Per facilitare la realizzazione della specifica algebrica è necessario introdurre un 
nuovo costruttore:
Build(Binary_tree, Elem, Binary_tree) → Binary_tree
Questo operatore costruisce un albero dato il valore del nodo (Elem) e i rispettivi 
sottoalberi sinistro e destro (primo e terzo parametro dell’operatore Build)
I costruttori nel nostro caso sono gli operatori: Create e Build, tutti gli altri 
operatori sono degli osservatori.

===== PAGE 16 =====
Soluzione Esercizio 3
declare: l, r: Binary_tree, e, e’: Elem
Add(Create(), e’) = Build(Create(), e’, Create())
Add(Build(l, e, r), e’) = if e’ < e then Add(l, e’) else Add(r, e’)
Left(Create()) = Create()
Left(Build(l, e, r)) = l
Right(Create()) = Create()
Right(Build(l, e, r)) = r
Data(Create()) = error
Data(Build(l, e, r)) = e
IsEmpty(Create()) = true
IsEmpty(Build(l, e, r)) = false
Contains(Create(), e’) = false
Contains(Build(l, e, r), e’) = if e’=e then true else (if e’<e then Contains(l, e’) else
  Contains(r, e’))

Esercizio 4
Definire la specifica algebrica del tipo Cursore. Un Cursore è una 
rappresentazione di una posizione sullo schermo. Le operazione definite sono:
●
Create: associa ad un cursore un’icona in una specifica posizione dello 
schermo
●
Position: restituisce la posizione corrente del cursore
●
Translate: sposta il cursore andando a modificare le coordinate attuali di una 
certa quantità
●
Change_Icon: che permette di cambiare l’icona associata al cursore
N.B. Come le immagini sono create sullo schermo e come viene associata 
un’immagine (icona) al cursore sono dettagli implementativi che non inseriamo 
nella specifica.
